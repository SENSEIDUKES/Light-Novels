## 2024-05-24 - Context-Aware Aria Labels for Card Components
**Learning:** Icon-only buttons and state-toggling buttons (like voice generation and visual aura manifestation) often lack descriptive `aria-label` attributes. Screen reader users face ambiguity when multiple similar cards render these identical icon-only buttons on the same page. Adding contextual properties like the entity's name (e.g., `Download scenery vista for ${loc.name}`) provides essential context.
**Action:** Always ensure dynamic, context-aware `aria-label` attributes are added to reusable list/card components when utilizing icon-only buttons.
